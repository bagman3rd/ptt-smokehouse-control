# Incident Response

Severity 1: production unavailable, tenant isolation concern, or data loss. Pause deployments, notify owner, preserve logs, restore from verified backup if required.
Severity 2: core workflow degraded. Assign an owner within 30 minutes and publish updates hourly.
Severity 3: non-blocking defect. Track in the next maintenance release.

Every incident requires timeline, root cause, corrective action, and a prevention test.
