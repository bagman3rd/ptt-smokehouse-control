# Build 5.4.0 — Documentation, Demo, and Sales Package

Build 5.4.0 turns the product from a strong internal pilot app into a more explainable and sellable product.

## Added pages

- `/tour` — guided workflow tour for operators, managers, and owners.
- `/sales` — sales one-pager explaining value, proof points, and weekly owner report structure.
- `/help` — expanded practical help docs and troubleshooting.
- `/demo` — clearer demo launch page describing the synthetic demo dataset.

## Demo changes

Demo Mode now creates 90 days of synthetic EOD history instead of four weeks. The larger sample makes Forecast Proof, Learning, Reports, and Data Quality more useful during sales demos.

## Help docs added

- How to add smokers.

## Sales package added

The `/sales` page gives Archer a direct product narrative:

- Reduce BBQ waste.
- Prevent sellouts.
- Standardize pitmaster decisions.
- Prove improvement.
- Train from real data.
- Run multiple restaurants.

## Remaining limitation

The app still needs real PTT operating data to make the 9+/10 forecast proof claim. The 90-day demo dataset is for demonstration only.
